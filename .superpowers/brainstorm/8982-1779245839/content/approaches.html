3 formas de mostrar tus fotos automáticamente

Todas usan Google Photos API con OAuth configurado una vez. La diferencia está en cómo y cuándo las fotos llegan a tu página.

📸 Google Photos → ⏰ Cron diario → 🗄 Supabase → 🌍 Page
Vercel Cron — cada noche a las 23:00
GET /api/sync-photos
→ lee fotos del día desde Google Photos
→ guarda URLs en Supabase
Europa page
→ lee de Supabase (rápido ⚡)

A — Sync diario con caché en Supabase

Un cron corre cada noche y guarda las fotos del día en tu base de datos. La página las carga instantáneamente desde Supabase.

Pros

  • Página carga rápido
  • Funciona si Google API está caída
  • Fácil de extender (captions, likes)

Contras

  • Fotos aparecen con hasta ~24h de delay
  • Más setup inicial
📸 Google Photos → 🌍 API Route → 🌍 Page
Cuando alguien abre la página de la ciudad
GET /api/photos?city=paris&date=2026-06-12
→ llama a Google Photos API live
→ retorna fotos al browser
⚡ Next.js caching puede mitigar latencia

B — Fetch directo a Google Photos (sin caché)

Cada vez que alguien abre la página, se consulta Google Photos en tiempo real. Sin base de datos, sin sync.

Pros

  • Fotos aparecen al instante (sin delay)
  • Setup más simple
  • Sin Supabase table extra

Contras

  • Más lento (depende de Google API)
  • Más caro en tokens de API
  • Fragil si Google API tiene issues
📸 Google Photos → 📂 Álbum del viaje → ⏰ Cron + IA → 🌍 Page
Google Photos tiene auto-álbumes por viaje ✨
🔸 Google Photos crea álbum automático
→ App lee ese álbum filtrado por fecha
→ Vision API / scoring selecciona mejores 10
→ Se muestran en la página

C — Álbum automático + selección inteligente con IA

Google Photos agrupa fotos por viaje automáticamente. La app lee ese álbum y usa Vision API o scoring básico (nitidez, no duplicados) para elegir las mejores 10.

Pros

  • Selección de "las mejores" fotos
  • Cero trabajo manual
  • Experiencia más curada sin curar

Contras

  • Más complejo de implementar
  • Costo extra (Vision API)
  • IA puede equivocarse en selección

💡 Mi recomendación: Opción A — simple, confiable, y la página carga rápido. La Opción B es un buen segundo si querés fotos en tiempo real. La C es la más "wow" pero más compleja.
