Opciones de integración con Google Photos

Tres caminos para conectar tu Next.js app con Google Photos — distintos niveles de complejidad y control.

googleapis (npm) OAuth2 manual
// setup una vez:
GOOGLE_REFRESH_TOKEN=xxx → Vercel env var
// en el cron diario:
oauth2Client.setCredentials({refresh_token})
fetch photos by date → score → Supabase

A — googleapis directo (recomendado)

Usás el paquete oficial de Google. OAuth2 se configura una vez: generás el refresh token antes del viaje y lo guardás como env var en Vercel.

Pros

  • Sin dependencias externas
  • Control total
  • Gratis
  • Código en el mismo repo

Contras

  • Hay que generar el refresh token manualmente (10 min de setup)
  • Un poco más de código
Composio OAuth gestionado Free tier
// Composio maneja todo el OAuth por vos:
composio.tools.googlephotos.searchMedia({
dateFilter: { startDate, endDate }
})
// 13 tools listos, sin manejar tokens

B — Composio (OAuth gestionado)

Composio maneja OAuth, refresh tokens y scopes. Tiene 13 acciones listas para Google Photos. El setup es más simple, pero agrega dependencia a un servicio externo.

Pros

  • No manejás refresh tokens
  • Setup muy rápido
  • 13 tools listos

Contras

  • Dependencia externa (si Composio falla, falla tu app)
  • Tus fotos pasan por sus servidores
  • Pricing puede cambiar
google-photos-mcp Open source
// Corre como servidor local MCP
// savethepolarbears/google-photos-mcp
✅ Picker API (post-deprecación Mar 2025)
✅ Search by date, location, category
⚠️ Diseñado para uso local (Claude Desktop)
⚠️ No corre directo en Vercel

C — google-photos-mcp (referencia)

MCP server open source que ya implementa la Picker API y la Library API. No corre en Vercel, pero su código TypeScript es perfecta referencia para implementar el API route propio.

Pros

  • Código TypeScript de referencia de calidad
  • Ya maneja la post-deprecación de scopes (Mar 2025)
  • Podés usar su patrón OAuth

Contras

  • No se embebe en Vercel directamente
  • Solo sirve como referencia
⚠️ Nota importante — Deprecación de scopes (Marzo 2025)
Google deprecó algunos scopes del Library API. Para uso personal (tu propia cuenta), el scope photoslibrary.readonly sigue funcionando. La Opción A con googleapis es la ruta más directa y segura.

💡 Recomendación: Opción A — googleapis directo. Control total, sin dependencias externas, costo $0. El código de google-photos-mcp (C) sirve como referencia de implementación.
